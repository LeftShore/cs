  • 前缀规范

    通过统一的容易理解和书写的前缀来代替以前常用的css hack，来提高代码质量的一种方法。

IE 增强

图例: 稳定支持 正在开发 有支持计划 不支持

  • 选择器语法增强
    • 交互选择器 (:hover,:focus,:active)
    • 状态选择器 (:target,:enabled,:disabled,:checked)
    • 属性选择器 ([attr="value"],[attr^="prefix"],[attr$="postfix"],[attr*="any"],[attr~="word"],)
    • 结构选择器 (:empty,:first-child,:last-child,nth-child(nth/odd/even),nth-last-child(nth),nth-of-(last)?-type(nth)...)
    • 选择器组合 (.classA.AndClassB, P>C, P+N, P~N, E:not(s))
  • 其他功能类增强
    • 半透明支持(opacity)

      用户可以直接书写CSS3 opacity属性

    • 变形增强(transform)

      位移参数尚未修正

    • 动画增强(transition)

      需要明确指定需要变化的 transition-property

    • 线性渐变(linear-gradient)

      目前采用filter实现，只能两个颜色，水平和垂直两个方向(不如CSS3强大)，后续可能尝试生成图片文件的方式

    • 阴影增强(box-shadow)

      计划用filter实现，不支持内阴影

    • 圆角增强(border-radius)

      目前采用filter＋vml实现，性能不佳，后续可能尝试动态生成图片文件的方式

    • RGBA 背景色(background:#11223355)

      通过filter渐变背景实现，很快会支持

    • PNG24 Alpha 背景图(background-image:xx.png)

      需要nodejs读取png源码，难度不搭，很快会支持

    • 溢出控制(max/min-width/height)

      无风险，很快会支持

    • 固定定位(position:fixed)

      无风险，很快会支持，可能有附加规范

    • 组合宽高增强(width:calc(50%-2px))

      看需求确定何时实现

    • 图形边框(border-image)

      有一定难度
