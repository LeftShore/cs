选择器语法增强

图例: IEn稳定支持 IEn正在开发 IEn有支持计划 IEn不支持

选择器支持相对完善，css3常用选择器基本支持。

例外细节

需求因素不计划支持的有:
namespace, :root, :lang(), @page :first |:right|:first ,

技术因素无法支持的有:
:before, :after, ::before, ::after, ::first-letter, ::first-line, ::selection, ::value, ::choices, ::repeat-item, ::repeat-index, ::first

时间因素尚未支持的有:
:target, :not(s), :indeterminate, :default, :valid, :invalid, :in-range, :out-of-range, :required, :optional, :read-only, :read-write

状态选择器

    交互状态

    一系列CSS3伪属性的支持，适当使用对提高交互体验作用明显，也可以配合动画功能制作CSS3动画。

    对效率的影响主要体现在可以通过简单的CSS申明完成以前需要多浏览器平台兼容的复杂脚本。

    • 鼠标悬浮效果:(:hover) IE6

      IE7开始原生支持。

      可用于下拉菜单，富文本提示信息，选项卡(Tab)/翻页组件等。也可用于CSS3动画的触发。

    • 焦点效果:(:focus) IE6 IE7

      IE8开始原生支持。

      定制焦点效果，增强输入控件和具有tabIndex属性的元素的互体验。也可用于触发CSS3动画。

    • 激活效果:(:active) IE6 IE7

      IE8开始原生支持。

      定制活动元素效果，改善交互体验。也可用于辅助制作选项卡(Tab)/翻页组件，也可触发CSS3动画。

      • 示例：dynamic.html

    UI/Hash状态

    • UI状态：(:checked/disabled/enabled) IE6 IE7 IE8

      IE9开始原生支持。

      控制元素在选中(用于：checkbox/radio)/禁用(disabled="disabled")时的样式控制。

    • Hash状态样式：(:target) IE6 IE7 IE8

      IE9开始原生支持。

      页内跳转的样式定制(#id)。

      • 示例：status.html

结构选择器

用于对DOM树中位置的自省，可取代部分模板逻辑和冗余的结构代码（如，nav-first,nav-last 等吃力的废话）

  • 首位元素(:first-child) IE6

    IE7开始原生支持。

    常用于菜单，按钮等ui组合的首尾显示元素匹配，您大可不必在模板中一堆繁琐的判断，其实CSS3一个伪属性搞定。

  • 末位元素(:last-child) IE6 IE7 IE8

    IE9开始原生支持。

    常用于菜单，按钮等ui组合的首尾显示元素匹配，您大可不必在模板中一堆繁琐的判断，其实CSS3一个伪属性搞定。

  • 单双行差异(:nth-child(odd),:nth-child(even)) IE6 IE7 IE8

    IE9开始原生支持。

    常用于信息列表的隔行差异化显示

  • 位置表达式：(nth-child(n),nth-last-child(n) IE6 IE7 IE8

    IE9开始原生支持。

    • list自定义序号，其他复杂顺序/关联匹配

      第几个元素，没必要在模板中申明class，这些都是CSS匹配的工作

  • 同类位置表达式：(nth-of-type(n),nth-last-of-type(n) IE6 IE7 IE8

    IE9开始原生支持。

  • 其他位置表达式:(:first-of-type,:last-of-type,:only-child,:only-of-type) IE6 IE7 IE8

    IE9开始原生支持。

  • 空节点:(:empty) IE6 IE7 IE8

    IE9开始原生支持。

    当某个元素没有任何子节点的时候生效

    • 示例：structural.html

组合选择器

  • IE6多class组合(.classA.andClassB) IE6

    IE6开始原生支持。

    IE6 CSS功能的落后也许会让你很无赖，比如他不支持直接子节点的样式匹配(>); 不支持相邻子节点的关系控制(~+)； 不支持同元素同时具有多个class时的样式控制。 等等

    如果系统能帮您抹平这些差异，我想应该是个很让人心情愉快的事情。

  • 紧邻次节点组合(previous+next_1) IE6

    IE6开始原生支持。

  • 次节点组合(previous~next_1_n) IE6

    IE6开始原生支持。

  • 排除选择器(:not(s)) IE6 IE7 IE8

    IE9开始原生支持。
